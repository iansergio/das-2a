# ADR-003: Seleção de biblioteca Python para conexão com Azure SQL Database

**Status:** Aceito | **Data:** 2026-05-27 | **Autores:** Equipe de Desenvolvimento

## Contexto

As Azure Functions responsáveis pela extração dos dados do ERP precisam se conectar ao Azure SQL Database via Python. Duas bibliotecas são amplamente utilizadas para esse fim: pyodbc e SQLAlchemy.

Foi conduzido um POC comparando o desempenho das duas abordagens em functions equivalentes — mesma lógica de extração, diferindo apenas na biblioteca de conexão. Além do desempenho, foi considerado o histórico da equipe com cada tecnologia.

## Decisão

Foi definido o uso de SQLAlchemy como biblioteca padrão para conexão com o Azure SQL Database nas Azure Functions do projeto.

## Evidências
Biblioteca | Execução 1 | Execução 2 | Média
---------- | ---------- | ---------- | -----
pyodbc     | 516 ms     | 461 ms     | 488 ms
SQLAlchemy | 747 ms     | 603 ms     | 675 ms

## Consequências

(+) (+) Equipe já familiarizada com a biblioteca — menor risco e curva de aprendizado

(+) Maior abstração — queries podem ser escritas com ORM ou SQL puro

(+) Maior portabilidade caso haja mudança de banco de dados no futuro

(-) Overhead adicional em relação ao pyodbc 

(-) Mais dependências no pacote da Azure Function

## Alternativas rejeitadas
**pyodbc:** melhor desempenho (aproximadamente 27,6% a mais) nos testes, porém sem experiência prévia da equipe e com menos abstrações para manutenção futura.
